# Brain Dump Beta User Interview Guide

Use this guide for watched first runs and follow-up calls. The goal is to learn whether Brain Dump creates trust and
repeat use, not to persuade the tester.

## Before The Run

- Confirm the tester understands Brain Dump is beta software.
- Confirm they know email sending is not enabled.
- Ask what they currently use for tasks, calendar, projects, and follow-ups.
- Ask what usually happens to their messy notes today.
- Ask them to bring a real but non-sensitive brain dump.

## During The Run

Observe without explaining unless they are blocked.

Watch for:

- Whether they understand preview mode.
- Whether Google connection feels trustworthy.
- Whether the review step is clear.
- Whether they notice they can remove a planned action.
- Whether calendar ambiguity feels safe or frustrating.
- Whether they can find created Tasks and Calendar events afterward.

## Core Questions

- What did you expect Brain Dump to do before you used it?
- What felt immediately useful?
- What felt risky, confusing, or wrong?
- Did the review step make you more comfortable?
- Which output would you keep, edit, or delete?
- Would you use this again next week?
- What would make this worth paying for later?

## Scoring

| Area | 1 | 3 | 5 |
| --- | --- | --- | --- |
| Setup clarity | Needed founder help | Minor friction | Self-serve |
| Output trust | Would not create | Would create some | Would create most |
| Review confidence | Missed controls | Understood after prompting | Understood immediately |
| Repeat intent | No | Maybe | Yes |
| Support load | Blocked | One question | No help needed |

## What Not To Ask

- Do not ask for Google passwords, OAuth tokens, or private screenshots.
- Do not ask leading questions that praise the product.
- Do not ask users to share sensitive task or calendar content.
- Do not promise features outside `/roadmap`.

## After The Run

- Ask them to submit `/feedback`.
- Mark feedback status in `/operator`.
- Add observed blocker category to the private tracker.
- Update the launch decision record if the run changes expansion readiness.
